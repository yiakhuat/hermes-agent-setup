iCloud Calendar MCP Tool

Connects to Apple's CalDAV API to manage iCloud calendar events directly from Hermes chat.

File Location

/root/icloud_calendar_mcp.py

Configuration in config.yaml

mcp_servers:
  icloud-calendar:
    command: /root/hermes-venv/bin/python3
    args:
    - /root/icloud_calendar_mcp.py
    timeout: 60
    connect_timeout: 60
    env:
      ICLOUD_EMAIL: your@icloud.com
      ICLOUD_PASSWORD: your-app-specific-password
Important: Use an App-Specific Password from appleid.apple.com → Security → App-Specific Passwords. Never use your regular Apple ID password.

MCP Tools Exposed

Tool NameDescription
mcp_icloud_calendar_list_eventsList upcoming calendar events
mcp_icloud_calendar_create_eventCreate a new calendar event
mcp_icloud_calendar_update_eventUpdate an existing event by UID

Example Usage

What's on my Family calendar this week?
Create a meeting called "Team Sync" on May 10 from 9am to 10am
Update the Coffee with Karin event to May 13

Job Scout MCP Tool

Searches Adzuna and JSearch job boards, filters results against saved preferences, deduplicates listings, and delivers a clean ranked digest. Set preferences once — remembered forever.

Architecture

User asks Hermes → job_scout_mcp.py ↓ Load job_profile.json ↓ ┌────────────┴────────────┐ ↓ ↓ Adzuna API JSearch API (250 req/mo) (200 req/mo) ↓ ↓ └────────────┬────────────┘ ↓ Filter → Rank → Deduplicate ↓ Save to seen_jobs.json ↓ Return digest to Hermes

Persistent Storage

FilePurpose
~/hermes/tools/job_scout/job_profile.jsonSaved preferences — loaded on every search
~/hermes/tools/job_scout/seen_jobs.jsonDeduplication cache — prevents repeat listings

Current Job Profile

{
  "job_titles": ["Technical Project Manager", "Localization Manager"],
  "locations": ["remote", "hybrid"],
  "min_salary": 140000,
  "experience": "senior",
  "industries": ["software", "consulting"],
  "max_days_old": 7
}

Configuration in config.yaml

mcp_servers:
  job-scout:
    command: /root/hermes-venv/bin/python3
    args:
    - /root/job_scout_mcp.py
    timeout: 60
    connect_timeout: 60
    env:
      ADZUNA_APP_ID: your_adzuna_app_id
      ADZUNA_APP_KEY: your_adzuna_app_key
      RAPIDAPI_KEY: your_rapidapi_key

Telephony Agent

Call your Twilio toll-free number from any phone and speak directly to Hermes AI. Supports English and Chinese (Mandarin).

Architecture

Incoming Call ↓ Twilio (toll-free number) ↓ Hetzner VPS :5001 ↓ hermes_telephony.py (Flask) ↓ hermes chat -q (CLI) ↓ DeepSeek AI ↓ Twilio TTS (Polly.Joanna / Polly.Zhiyu) ↓ Caller hears response

Installation

# Install dependencies
/root/hermes-venv/bin/python3 -m pip install twilio flask

# Deploy service
cp hermes-telephony.service ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable hermes-telephony
systemctl --user start hermes-telephony

Troubleshooting

Service Health Checks

systemctl --user status hermes-gateway
systemctl --user status hermes-dashboard
systemctl --user status hermes-telephony
curl http://localhost:5000/api/health
curl http://localhost:5001/health

Check Logs

tail -30 ~/.hermes/logs/agent.log
tail -20 ~/.hermes/logs/mcp-stderr.log

Validate config.yaml

/root/hermes-venv/bin/python3 -c "
import yaml
try:
    yaml.safe_load(open('/root/.hermes/config.yaml'))
    print('OK - no errors')
except yaml.YAMLError as e:
    print('ERROR:', e)
"

Common Errors

ErrorFix
NoneType object is not iterableRun hermes model and switch to DeepSeek provider
Failed to parse config.yamlCheck for tab characters or duplicate keys in YAML
MCP server registered 0 toolsVerify API keys in config.yaml env section
File not found: tts_*.mp3Clear cache: rm -f ~/.hermes/audio_cache/tts_*
Session DB: file is not a databaseRemove: rm -f ~/.hermes/state.db* and restart
WhatsApp not respondingClear sessions: rm -rf ~/.hermes/sessions/* and restart gateway
